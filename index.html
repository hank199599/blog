第五章 淺談語音對話介面

發表於 2021-01-23 分類於 Google Assistant

語音對話介面 (Voice User Interface)

  • 單輪對話：一問一答就結束對話
  • 多輪對話：一問一答的同時衍生出新的問題和新的回答，從而無限接近用戶的真實訴求

傳統的語音互動都是單輪對話，而隨着技術的發展，多輪對話成爲語音互動的趨勢。

VUI設計工具

  • 示例對話：爲VUI挑選最常見的使用場景，爲這些場景寫一系列最優路徑的示例對話以及異常情況的示例對話。

  • 視覺原型圖：視覺原型圖可將用戶體驗可視化，結合VUI，讓用戶產生更完整的視聽體驗。

  • 流程圖：設計使用者在與其互動時的操作的可能流程

設計概念

確定策略：控制式還是對話式。
確認策略時可參考：

  1. 錯誤的後果是什麼？
  2. 系統將以什麼形式進行反饋？
  3. 以什麼形式來確認？

命令：控制模式、對話模式

目前主流的是命令控制模式，而隨着AI的發展，更流暢自然的輪流對話設計模式將變得更普遍。控制模式最簡單的例子是，當我們要和Siri對話時，我們必須長按Home鍵（push-to-talk），纔可喚醒Siri。而對話模式則不會有控制鍵告訴機器用戶要開始說話了，而是用更自然的對話技巧進行話輪轉換；比如問一個問題，使用眼神交流，給出明確的提示等…… 需要強調的是不要強迫用戶在不合適的情況下進行話輪轉換。讓VUI更加人性化或更具吸引力的方式是使用對話標識。對話標識能對用戶的信息有正面反饋，或者是提問前讓用戶能有所預期。對話標識包括：時間線、接受回饋、積極反饋等

置信度閾值：VUI主要通過語音來反饋結果，確認信息對於體驗非常重要，要做到這一點需要使用置信度閾值。使用三級置信度時，系統將一定的閾值內以明確的形式確認信息，若是信息置信度小於45％，則系統會通過顯性確認信息。若是信息置信度大於80％，則系統將以隱性置信度來確認。

顯性確認：需要強制用戶確認信息，比如是否確定支付某一筆金額時，必須用戶確認纔可以，而且必須留有歷史語音記錄。

隱性確認：用戶只需要接受信息，但無需強制確認，若是隱性確認，則允許用戶撤銷；

非語言式確認：僅需行動反饋，無需口頭響應。就比如你說“打開窗簾”，此時智能家居系統就直接把窗簾打開了，無需再經過語音系統來反覆確認。另外非語言式確認包含了特定音樂來反饋結果，比如洗衣機洗完衣服，會用一段美妙的音樂告知用戶…..

通用確認：通用確認並不需要用戶確認具體項目，而是開放式的聊天，從中我們可以瞭解用戶的心情和狀態等。這類反饋需要一些通用性的回答。

視覺確認：通常用戶一次性不能記住超過大約7個聽覺項目，語音交互稍縱即逝，對於用戶的認知提出了新挑戰，可視化列表可降低用戶的認知難度。使用屏幕的另一個好處，我們能讓用戶快速確認某一件事情，但語音確認就會遇到置信度閾值問題。

異常處理：語音交互因環境噪聲或用戶聲音的輕重，都有可能導致系統出錯，VUI出錯的情況有：未檢測到語音信息；檢測到了語音，但未識別出結果；語音被正確識別了，但系統不能處理這些信息的反饋；部分語音信息識別出錯。

記錄對話的上下文，爲後續的語音輸出提供信息和線索。儘量多的記錄用戶和語音的交互信息，這些信息可以幫助系統瞭解用戶是誰，以及用戶的一些角色行爲特徵。

第三章 建立Dialogflow 專案

發表於 2020-11-19 更新於 2020-11-20 分類於 Google Assistant

Dialogflow

是一項屬於Google的開發工具，提供基於 自然語言對話(NLU) 的人機互動技術。

如何偕同Google助理運作的?

您可能想知道Google助理如何解析用戶輸入的語義（如語音）。
這是通過自然語言理解（NLU）來完成的，它使Google的軟體能夠識別語音中的單詞。

對於開發者自己的Action來說，
Dialogflow簡化了理解用戶輸入，從輸入中提取關鍵詞和短語以用來回應請求的意圖。
開發者可以在Dialogflow定義這一切的運作方式。

建立DialogFlow專案

  1. 前往DialgoFlow主控台
    點擊位於左側選單上方的 [Create Agent],
  2. 填寫Agent名稱以及要使用的Google Project
    接著點擊位於畫面右上方的[Create]
  3. 待上述步驟完成後，你就已經完成專案設定了!

下一步

在下一章中，我們會再次前往Actions On Google平台。
實際體驗看看加入Dialogflow偕同運作的Actions。

第二章 建立Action on Google專案

發表於 2020-11-17 分類於 Google Assistant

概述Action專案


它被用來專載你的Action，
無論是建構對話式Action或智慧家庭的介接都需要經過這一步驟。

Actions目錄概觀

是您Action的容器，
裝載與您Action相關的數據（名稱，描述，類別）。這些內容會構成您Action的目錄內容：

  1. Action 名稱
  2. 內容描述
  3. Action使用類別
  4. 使用範例
  5. 適用裝置

建立前提：檢查你的Google權限設定

  1. 前往「活動控制項」頁面
  2. 登入你的Google帳號
  3. 請確認以下權限都已經啟用：
    網路和應用程式活動
  • 包括 Chrome 歷史記錄以及採用 Google 服務的網站、應用程式和裝置中的活動記錄
  • 包括音訊記錄

若你未完成上述設定，可能會在後續的開發中碰到錯誤。
請確認你已經完成這些設定後再繼續操作喲!

登入Actions On Google 控制台

  1. 點擊這個連結進入Actions On Google 控制台

  2. 點擊[New Project]按鈕

  3. 在跳出選單中輸入以下資訊：

    • Project Name
    • 選取 Action與用戶互動預設的使用語言
    • 選取 你所在的國家/地區
      待上述資訊都設定完畢後，按右下方的[Create Project]

  4. 進入初始化選單，在這裡請選擇「Custom」

  5. 進入進階選單，在這裡請選擇位在選單最下方的「build your Action with Dialogflow」

  6. 恭喜你，大功告成啦!
    請前往下一個章節繼續進行下一個專案設定!

第一章 基礎運作原理

發表於 2020-11-15 更新於 2020-11-17 分類於 Google Assistant

基礎運作方式

Google助理作為中介腳色，
將使用者輸入的語句以傳給第三方的Action。
待第三方Action處理完生成對應回應後再藉由Google助理呈現給使用者。

與第三方Action對話的過程中，除非Action自動退出對話或觸發對應Intent。
是無法離開與第三方Action的互動的。

呼叫Action的方法

根據官方說明頁面，在中文(台灣)的使用情境下支援以下呼叫方式：

觸發動作名稱的基本動詞：

  • 叫
  • 對話
  • 對談
  • 詢問
  • 說話
  • 講話

用戶能指示Google助理存取您的操作的示例短語：

  • 幫我叫$name
  • 能幫我叫下 $name 嗎
  • 我想和 $name 對話
  • 我可以和 $name 進行對談嗎
  • 詢問 $name
  • 我想找 $name 說話
  • 我要跟 $name 說話
  • 我要跟 $name 講話

調用Action的方式

呼叫Google助理調用Action的方式，
依據用戶是否明白指出調用的Action而分為兩種調用方式：

顯式調用(Explicit Invocation)

使用者清楚地向Google助理表達欲使用的Action為何者

未指定特定動作

一般情況下，若使用者未向Action指定想進行的操作為何。
Google會將使用者導入開發者預定的預設歡迎畫面(Default Welcome Intent)

指定特定動作 ：語音版捷徑

用戶也可以選擇在調用同時添加一個調用短語，
這方式得以將使用者直接帶到他們所請求的意圖。
其語言架構如下所示：

用戶能指示Google助理存取您的操作的示例短語：

  • 叫$name $request_action
  • 跟 $name 說 $request_action
  • 問 $name $request_action

實際系統運作時的架構圖

隱式調用(Implicit invocation)


調用短語描述了Action的特定功能。
用戶調用Action時，可能會包含一個可以深度鏈接到特定功能之一的調用短語。
舉例來說，如果使用者想玩1A2B。
可能會直接向Google助理詢問。
而Google助理就會自Actions On Google平台尋找最類似的Action推薦使用者。
不過這項功能在中文版尚未啟用，只能在draft(草案)模式下試用。

加入Fulfillment後的運作方式

詳情參見：第五章 撰寫 DialogFlow Fulfillment


在DialogFlow的使用情境中，
開發者只能進行基本的回應而無法加入邏輯判斷等深度回應。
透過部屬於雲端Function，能讓Google助理進行更多元的功能。
例如：抓取API獲取資料、存取Database等操作

Google 助理簡介

發表於 2020-11-15 更新於 2021-01-23 分類於 Google Assistant

Google Assistant APP

Google助理是由Google所開發之智慧型個人助理，
在2018年10月16日於臺灣上線。其後在隔年三月開放台灣的第三方程式可以與其介接。
Google提供智慧家庭裝置以及語音應用程式與其進行介接，讓使用者能藉由語音進行便利地互動!

執行原理

  • Google助理：
    • 為Google開發的智慧型個人助理
    • 在第三方應用的角色是處理語音辨識及傳遞回應給使用者
  • DialogFlow：
    • 建構自然語言處理模型並訓練機器辨識使用者輸入的意圖(Intent)
    • 並作為中介把資訊傳遞給Fulfillment
  • Fulfillment：
    • 將來自DialogFlow的資訊進行分析再回應給使用者
    • 實作上是將Fulfillment部署到Firebase上，作為處理與回傳客製化回應的中介
