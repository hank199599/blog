第七章 介紹 Node.js 與 NPM套件

發表於 2021-01-23 更新於 2021-02-18 分類於 Google Assistant

有關 JavaSript

JavaScript是一種：

  1. 物件導向（Object-oriented programming, OOP）
  2. 弱型別（Week type）
  3. 腳本語言（Scripting language）

Node.js 基礎


簡而言之，這是一種用於服務器端編程的JavaScript免費開放源代碼跨平台，
可讓用戶快速構建網絡應用程序。

基礎

“Hello World”

宣告（Declarations）

JavaScript有三種宣告方式

  • var：宣告一個可隨意更改其內容的變數

  • let：宣告一個可隨意更改其內容的區塊區域變數

  • const：宣告一個只可讀取的不可變常數

    變數型態

  • 數字

    • 十進制 (decimal)
    • 二進制 (binary)
      以 0 為開頭並跟著一個大寫或小寫的英文字母 「B」 (0b 或 0B)
    • 八進制 (octal)
      以 0 為開頭。如果 0 後面的數字超出 0 到 7 這個範圍，將會被解析成十進制數值。
    • 十六進制 (hexadecimal)
      以 0 為開頭並跟著一個大寫或小寫的英文字母 「X」(0x 或 0X)。
    • 指數型態

  • 字串

  • 布林值

資料結構

  • 陣列
  • 字典
  • null
    變數要經過宣告，賦予 null ，才會形成 null 型態。
  • undefined
    從字面上就表示目前未定義，只要一個變數在初始的時候未給予任何值的時候，就會產生 undefined

大小判斷

  • 大於
  • 小於
  • 等於(==)
  • 精確等於(===)
  • 不等於(!=)
  • 精確不等於(!==)

邏輯判斷

  • if /else
    當條件成立的時候會執行 if 陳述式裡的程式，而不成立時則執行另外一個陳述式。
  • for
    創建一個循環，該循環由三個可選表達式組成，並用括號括起來並用分號分隔，然後是要在循環中執行的語句（通常是塊語句）。
  • while
    創建一個循環，該循環執行指定的語句，直到測試條件的值為假。執行該語句後評估條件，導致指定的語句至少執行一次。

    NPM套件

npm 全名為 Node Package Manager， 是 Node.js 的套件（package）管理工具。
安裝 npm 後，使用 npm install module_name 指令即可安裝新套件，維護管理套件的工作會更加輕鬆!

npm 可以讓 Node.js 的開發者，直接利用、擴充線上的套件庫（packages registry），加速軟體專案的開發。npm 提供很友善的搜尋功能，可以快速找到、安裝需要的套件，當這些套件發行新版本時，npm 也可以協助開發者自動更新這些套件。

npm 不僅可用於安裝新的套件，它也支援搜尋、列出已安裝模組及更新的功能。

第六章 設計對話流

發表於 2021-01-23 更新於 2021-01-24 分類於 Google Assistant

範例：詢問用戶喜歡的顏色

在這個範例裡，我們假設一個要蒐集使用者偏好顏色的資料集。
並透過語音助理來協助我們進行資料蒐集。
為此我們需要先假想使用者與之互動時可能的對話流，並加以進行改進。

於Dialogflow上實作

你可以透過上傳我預先做好的angent.zip 到你的DialogFlow專案，
搭配這篇文章服用能更快理解以下內容的操作!
專案頁面

建立顏色的數據集(Entities)

  1. 首先，透過左側的「Entities」選項切換到數據集頁面。
  2. 接著，點選右上角的「Create Entities」建立我們所需要的資料集
  3. 將這個資料集命名為「color」，

建立蒐集顏色的Intent(意圖)

  1. 切換回「Intents」頁面，建立一個新的Intent來客製化新的對話流程。
    點選畫面右上角的「Create Intent」。
  2. 將Intent名稱設定為「用戶輸入的顏色」，在「Training pharse」輸入一些用戶可能會說的話來訓練模型。 例如：我喜歡綠色

  3. 接著向下滾動頁面，我們要進一步設計這個Intent所給予的回應。
  • 設計「Response」，輸入「真巧，我也喜歡$color」
  • 根據我們先前的對話流設計，在取得用戶偏好的顏色後就會結束對話。
    因此「Set this intent as end of conversation」的開關要打開。
    如此一來，當用戶說出他喜歡的顏色後就會自動離開對話。

  1. 在上述操作完成後，點擊「Save」來儲存剛剛的設定。

修改不明回應的Intent(意圖)

  1. 回到展示所有Intent的頁面，現在會看到「用戶輸入的顏色」已經出現在列表中了!
    請點選「Default Fallback Intent」修改模型在碰到無關輸入時要進行的回應。
  2. 首先，把原本被填寫的回應清除。
  3. 接著，填上我們在對話流設計的範本中所預想的回應：「不好意思，請問你喜歡的顏色是甚麼?」

在 Google助理上試用

依照第四章 在Google Assistant上試用的步驟，於Actions On Google平台上試用。
可以發現到，它的確依據我們設計的對話流在進行互動!

進入下一個階段

你會注意到，在這裡所做的事情只是單純讓他具備雙向的對話能力，
卻無法有邏輯性地判斷使用者的輸入(例如：輸入是不是數字?)
因此，在接下來的章節將先介紹我們將使用的程式語言與對應的模組系統。

第五章 淺談語音對話介面

發表於 2021-01-23 更新於 2021-01-24 分類於 Google Assistant ， 語音使用者介面

語音對話介面 (Voice User Interface,VUI)

  • 單輪對話：一問一答就結束對話
  • 多輪對話：一問一答的同時衍生出新的問題和新的回答，從而無限接近用戶的真實訴求

傳統的語音互動都是單輪對話，而隨着技術的發展，多輪對話成爲語音互動的趨勢。

設計工具

  • 示例對話：爲VUI挑選最常見的使用場景，爲這些場景寫一系列最優路徑的示例對話以及異常情況的示例對話。

  • 視覺原型圖：視覺原型圖可將用戶體驗可視化，結合VUI，讓用戶產生更完整的視聽體驗。

  • 流程圖：設計使用者在與其互動時的操作的可能流程

設計概念

確定策略：控制式還是對話式。
確認策略時可參考：

  1. 錯誤的後果是什麼？
  2. 系統將以什麼形式進行反饋？
  3. 以什麼形式來確認？

命令

  • 控制模式：透過特定按鍵呼叫語音助理
  • 對話模式：使用更自然的對話技巧進行話輪轉換

置信度閾值

VUI主要通過語音來反饋結果，確認訊息對於體驗非常重要，要做到這一點需要使用置信度閾值。使用三級置信度時，系統將一定的閾值內以明確的形式確認訊息，若是訊息置信度小於45％，則系統會通過顯性確認訊息。若是訊息置信度大於80％，則系統將以隱性置信度來確認。

確認方式

  • 顯性確認：需要強制用戶確認訊息

  • 隱性確認：用戶只需要接受訊息，但無需強制確認

  • 非語言式確認：僅需行動反饋，無需口頭響應。例如：「打開窗簾」

  • 通用確認：通用確認並不需要用戶確認具體項目，而是開放式的聊天，從中我們可以瞭解用戶的心情和狀態等。這類反饋需要一些通用性的回答。

  • 視覺確認：使用螢幕讓用戶快速確認某件事情

異常處理

因環境噪聲或用戶聲音的輕重，導致系統出錯。
出錯的情況有：

  • 未檢測到語音訊息；檢測到了語音，但未識別出結果
  • 語音被正確識別了，但系統不能處理這些訊息的反饋
  • 部分語音訊息識別出錯。

第三章 建立Dialogflow 專案

發表於 2020-11-19 更新於 2020-11-20 分類於 Google Assistant

Dialogflow

是一項屬於Google的開發工具，提供基於 自然語言對話(NLU) 的人機互動技術。

如何偕同Google助理運作的?

您可能想知道Google助理如何解析用戶輸入的語義（如語音）。
這是通過自然語言理解（NLU）來完成的，它使Google的軟體能夠識別語音中的單詞。

對於開發者自己的Action來說，
Dialogflow簡化了理解用戶輸入，從輸入中提取關鍵詞和短語以用來回應請求的意圖。
開發者可以在Dialogflow定義這一切的運作方式。

建立DialogFlow專案

  1. 前往DialgoFlow主控台
    點擊位於左側選單上方的 [Create Agent],
  2. 填寫Agent名稱以及要使用的Google Project
    接著點擊位於畫面右上方的[Create]
  3. 待上述步驟完成後，你就已經完成專案設定了!

下一步

第一章 基礎運作原理

發表於 2020-11-15 更新於 2020-11-17 分類於 Google Assistant

基礎運作方式

Google助理作為中介腳色，
將使用者輸入的語句以傳給第三方的Action。
待第三方Action處理完生成對應回應後再藉由Google助理呈現給使用者。

與第三方Action對話的過程中，除非Action自動退出對話或觸發對應Intent。
是無法離開與第三方Action的互動的。

呼叫Action的方法

根據官方說明頁面，在中文(台灣)的使用情境下支援以下呼叫方式：

觸發動作名稱的基本動詞：

  • 叫
  • 對話
  • 對談
  • 詢問
  • 說話
  • 講話

用戶能指示Google助理存取您的操作的示例短語：

  • 幫我叫$name
  • 能幫我叫下 $name 嗎
  • 我想和 $name 對話
  • 我可以和 $name 進行對談嗎
  • 詢問 $name
  • 我想找 $name 說話
  • 我要跟 $name 說話
  • 我要跟 $name 講話

調用Action的方式

呼叫Google助理調用Action的方式，
依據用戶是否明白指出調用的Action而分為兩種調用方式：

顯式調用(Explicit Invocation)

使用者清楚地向Google助理表達欲使用的Action為何者

未指定特定動作

一般情況下，若使用者未向Action指定想進行的操作為何。
Google會將使用者導入開發者預定的預設歡迎畫面(Default Welcome Intent)

指定特定動作 ：語音版捷徑

用戶也可以選擇在調用同時添加一個調用短語，
這方式得以將使用者直接帶到他們所請求的意圖。
其語言架構如下所示：

用戶能指示Google助理存取您的操作的示例短語：

  • 叫$name $request_action
  • 跟 $name 說 $request_action
  • 問 $name $request_action

實際系統運作時的架構圖

隱式調用(Implicit invocation)


調用短語描述了Action的特定功能。
用戶調用Action時，可能會包含一個可以深度鏈接到特定功能之一的調用短語。
舉例來說，如果使用者想玩1A2B。
可能會直接向Google助理詢問。
而Google助理就會自Actions On Google平台尋找最類似的Action推薦使用者。
不過這項功能在中文版尚未啟用，只能在draft(草案)模式下試用。

加入Fulfillment後的運作方式

詳情參見：第五章 撰寫 DialogFlow Fulfillment


在DialogFlow的使用情境中，
開發者只能進行基本的回應而無法加入邏輯判斷等深度回應。
透過部屬於雲端Function，能讓Google助理進行更多元的功能。
例如：抓取API獲取資料、存取Database等操作

Google 助理簡介

發表於 2020-11-15 更新於 2021-01-23 分類於 Google Assistant

Google Assistant APP

Google助理是由Google所開發之智慧型個人助理，
在2018年10月16日於臺灣上線。其後在隔年三月開放台灣的第三方程式可以與其介接。
Google提供智慧家庭裝置以及語音應用程式與其進行介接，讓使用者能藉由語音進行便利地互動!

執行原理

  • Google助理：
    • 為Google開發的智慧型個人助理
    • 在第三方應用的角色是處理語音辨識及傳遞回應給使用者
  • DialogFlow：
    • 建構自然語言處理模型並訓練機器辨識使用者輸入的意圖(Intent)
    • 並作為中介把資訊傳遞給Fulfillment
  • Fulfillment：
    • 將來自DialogFlow的資訊進行分析再回應給使用者
    • 實作上是將Fulfillment部署到Firebase上，作為處理與回傳客製化回應的中介
