第四章 在Google Assistant上試用

發表於 2020-11-26 更新於 2021-01-23 分類於 Google Assistant

踏出第一步：修改對話內容

首先，點擊進入Default Welcome Intent，並更改系統預設的回應

刪除多餘的回應，最後只留下一個回應，並將其更改為：
「歡迎，你喜歡的顏色是什麼?」

進入正題，前往Actions On Google平台試用

  1. 點擊左側選單面板「Intergrations」，

  2. 開啟頁面後，點選裡面的「TEST」按鈕
  3. 接著系統會把你在Dialogflow上建立的模型上傳到 Actions On Google
  4. 載入完成後，映入眼簾的是 Actions On Google 測試頁面。
    請點選畫面左側的「我要跟我的測試應用程式說話」的按鈕啟用測試!
  5. 接著，就能看到剛剛撰寫的對話在Google助理的裝置上出現了!

第三章 建立Dialogflow 專案

發表於 2020-11-19 更新於 2020-11-20 分類於 Google Assistant

Dialogflow

是一項屬於Google的開發工具，提供基於 自然語言對話(NLU) 的人機互動技術。

如何偕同Google助理運作的?

您可能想知道Google助理如何解析用戶輸入的語義（如語音）。
這是通過自然語言理解（NLU）來完成的，它使Google的軟體能夠識別語音中的單詞。

對於開發者自己的Action來說，
Dialogflow簡化了理解用戶輸入，從輸入中提取關鍵詞和短語以用來回應請求的意圖。
開發者可以在Dialogflow定義這一切的運作方式。

建立DialogFlow專案

  1. 前往DialgoFlow主控台
    點擊位於左側選單上方的 [Create Agent],
  2. 填寫Agent名稱以及要使用的Google Project
    接著點擊位於畫面右上方的[Create]
  3. 待上述步驟完成後，你就已經完成專案設定了!

下一步

在下一章中，我們會再次前往Actions On Google平台。
實際體驗看看加入Dialogflow偕同運作的Actions。

第二章 建立Action on Google專案

發表於 2020-11-17 分類於 Google Assistant

概述Action專案


它被用來專載你的Action，
無論是建構對話式Action或智慧家庭的介接都需要經過這一步驟。

Actions目錄概觀

是您Action的容器，
裝載與您Action相關的數據（名稱，描述，類別）。這些內容會構成您Action的目錄內容：

  1. Action 名稱
  2. 內容描述
  3. Action使用類別
  4. 使用範例
  5. 適用裝置

建立前提：檢查你的Google權限設定

  1. 前往「活動控制項」頁面
  2. 登入你的Google帳號
  3. 請確認以下權限都已經啟用：
    網路和應用程式活動
  • 包括 Chrome 歷史記錄以及採用 Google 服務的網站、應用程式和裝置中的活動記錄
  • 包括音訊記錄

若你未完成上述設定，可能會在後續的開發中碰到錯誤。
請確認你已經完成這些設定後再繼續操作喲!

登入Actions On Google 控制台

  1. 點擊這個連結進入Actions On Google 控制台

  2. 點擊[New Project]按鈕

  3. 在跳出選單中輸入以下資訊：

    • Project Name
    • 選取 Action與用戶互動預設的使用語言
    • 選取 你所在的國家/地區
      待上述資訊都設定完畢後，按右下方的[Create Project]

  4. 進入初始化選單，在這裡請選擇「Custom」

  5. 進入進階選單，在這裡請選擇位在選單最下方的「build your Action with Dialogflow」

  6. 恭喜你，大功告成啦!
    請前往下一個章節繼續進行下一個專案設定!

第一章 基礎運作原理

發表於 2020-11-15 更新於 2020-11-17 分類於 Google Assistant

基礎運作方式

Google助理作為中介腳色，
將使用者輸入的語句以傳給第三方的Action。
待第三方Action處理完生成對應回應後再藉由Google助理呈現給使用者。

與第三方Action對話的過程中，除非Action自動退出對話或觸發對應Intent。
是無法離開與第三方Action的互動的。

呼叫Action的方法

根據官方說明頁面，在中文(台灣)的使用情境下支援以下呼叫方式：

觸發動作名稱的基本動詞：

  • 叫
  • 對話
  • 對談
  • 詢問
  • 說話
  • 講話

用戶能指示Google助理存取您的操作的示例短語：

  • 幫我叫$name
  • 能幫我叫下 $name 嗎
  • 我想和 $name 對話
  • 我可以和 $name 進行對談嗎
  • 詢問 $name
  • 我想找 $name 說話
  • 我要跟 $name 說話
  • 我要跟 $name 講話

調用Action的方式

呼叫Google助理調用Action的方式，
依據用戶是否明白指出調用的Action而分為兩種調用方式：

顯式調用(Explicit Invocation)

使用者清楚地向Google助理表達欲使用的Action為何者

未指定特定動作

一般情況下，若使用者未向Action指定想進行的操作為何。
Google會將使用者導入開發者預定的預設歡迎畫面(Default Welcome Intent)

指定特定動作 ：語音版捷徑

用戶也可以選擇在調用同時添加一個調用短語，
這方式得以將使用者直接帶到他們所請求的意圖。
其語言架構如下所示：

用戶能指示Google助理存取您的操作的示例短語：

  • 叫$name $request_action
  • 跟 $name 說 $request_action
  • 問 $name $request_action

實際系統運作時的架構圖

隱式調用(Implicit invocation)


調用短語描述了Action的特定功能。
用戶調用Action時，可能會包含一個可以深度鏈接到特定功能之一的調用短語。
舉例來說，如果使用者想玩1A2B。
可能會直接向Google助理詢問。
而Google助理就會自Actions On Google平台尋找最類似的Action推薦使用者。
不過這項功能在中文版尚未啟用，只能在draft(草案)模式下試用。

加入Fulfillment後的運作方式

詳情參見：第五章 撰寫 DialogFlow Fulfillment


在DialogFlow的使用情境中，
開發者只能進行基本的回應而無法加入邏輯判斷等深度回應。
透過部屬於雲端Function，能讓Google助理進行更多元的功能。
例如：抓取API獲取資料、存取Database等操作

Google 助理簡介

發表於 2020-11-15 更新於 2020-11-19 分類於 Google Assistant

Google Assistant APP

Google助理是由Google所開發之智慧型個人助理，
在2018年10月16日於臺灣上線。其後在隔年三月開放台灣的第三方程式可以與其介接。
Google提供智慧家庭裝置以及語音應用程式與其進行介接，讓使用者能藉由語音進行便利地互動!

執行原理

  • Google助理：
    • 為Google開發的智慧型個人助理
    • 在第三方應用的角色是處理語音辨識及傳遞回應給使用者
  • DialogFlow：
    • 建構自然語言處理模型並訓練機器辨識使用者輸入的意圖(Intent)
    • 並作為中介把資訊傳遞給Fulfillment
  • Fulfillment：
    • 將來自DialogFlow的資訊進行分析再回應給使用者
    • 實作上是將Fulfillment部署到Firebase上，作為處理與回傳客製化回應的中介
