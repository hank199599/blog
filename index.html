第一章 基礎運作原理

發表於 2020-11-15 更新於 2020-11-16 分類於 Google Assistant

基礎運作方式

Google助理作為中介腳色，
將使用者輸入的語句以傳給第三方的Action。
待第三方Action處理完生成對應回應後再藉由Google助理呈現給使用者。

與第三方Action對話的過程中，除非Action自動退出對話或觸發對應Intent。
是無法離開與第三方Action的互動的。

呼叫Action的方法

根據官方說明頁面，在中文(台灣)的使用情境下支援以下呼叫方式：

觸發動作名稱的基本動詞：

  • 叫
  • 對話
  • 對談
  • 詢問
  • 說話
  • 講話

用戶能指示Google助理存取您的操作的示例短語：

  • 幫我叫$name
  • 能幫我叫下 $name 嗎
  • 我想和 $name 對話
  • 我可以和 $name 進行對談嗎
  • 詢問 $name
  • 我想找 $name 說話
  • 我要跟 $name 說話
  • 我要跟 $name 講話

調用Action的方式

呼叫Google助理調用Action的方式，
依據用戶是否明白指出調用的Action而分為兩種調用方式：

顯式調用(Explicit Invocation)

使用者清楚地向Google助理表達欲使用的Action為何者

未指定特定動作

一般情況下，若使用者未向Action指定想進行的操作為何。
Google會將使用者導入開發者預定的預設歡迎畫面(Default Welcome Intent)

指定特定動作 ：語音版捷徑

用戶也可以選擇在調用同時添加一個調用短語，
這方式得以將使用者直接帶到他們所請求的意圖。
其語言架構如下所示：

用戶能指示Google助理存取您的操作的示例短語：

  • 叫$name $request_action
  • 跟 $name 說 $request_action
  • 問 $name $request_action

實際系統運作時的架構圖

隱式調用(Implicit invocation)


調用短語描述了Action的特定功能。
用戶調用您的操作時，可能會包含一個可以深度鏈接到特定功能之一的調用短語。
舉例來說，如果使用者想玩1A2B。
可能會直接向Google助理詢問。
而Google助理就會自Actions On Google平台尋找最類似的Action推薦使用者。
不過這項功能在中文版尚未啟用，只能在draft(草案)模式下試用。

加入Fulfillment後的運作方式

詳情參見：第五章 撰寫 DialogFlow Fulfillment


在DialogFlow的使用情境中，
開發者只能進行基本的回應而無法加入邏輯判斷等深度回應。
透過部屬於雲端Function，能讓Google助理進行更多元的功能。
例如：抓取API獲取資料、存取Database等操作

Google 助理簡介

發表於 2020-11-15 更新於 2020-11-16 分類於 Google Assistant

Google Assistant APP

Google助理是由Google所開發之智慧型個人助理，
在2018年10月16日於臺灣上線。其後在隔年三月開放台灣的第三方程式可以與其介接。
Google提供智慧家庭裝置以及語音應用程式與其進行介接，讓使用者能藉由語音進行便利地互動!

執行原理

  • Google助理：
    • 為Google開發的智慧型個人助理
    • 在第三方應用的角色是處理語音辨識及傳遞回應給使用者
  • DialogFlow：
    • 建構自然語言處理模型並訓練機器辨識使用者輸入的意圖(Intent)
    • 並作為中介把資訊傳遞給Fulfillment
  • Fulfillment：
    • 將來自DialogFlow的資訊進行分析再回應給使用者
    • 實作上是將Fulfillment部署到Firebase上，作為處理與回傳客製化回應的中介
