第八章 撰寫 DialogFlow Fulfillment

發表於 2021-01-23 更新於 2021-04-04 分類於 Google Assistant 閱讀次數：

在了解簡易的JavaScript開發流程後，
我們要開始編寫第一個能夠利用邏輯判斷的Untent了!

在開始前：為何要採用Fulfillment來撰寫回應?

在Dialogflow上，我們能做的設計被侷限在單純的回應。
無法進行更進一步的互動，像是：

  1. 執行邏輯判斷
  2. 抓取API
  3. 存取Database
  4. Bullet five

因此，透過接入Fulfillment。
能使回應之設計更具多樣性，達到更貼近一般生活上的對話互動。

開啟內建編輯器

  1. 首先，前往「Fulfillment」分頁。
    開啟Inline Editor的功能，我們將透過他於GCP上建立雲端函式。
    並以此撰寫程式來客製化回應。

  2. 開啟該功能後，系統會提示需要啟用GCP的付費功能。
    請點擊「OPEN CLOUD CONSOLE」繼續操作。

  3. 跟著系統的導引建立帳單帳戶，並將其綁訂到這個專案上

  4. 現在，你已經成功開啟我們所需要的雲端編輯器啦!

額外補充

在Fulfillment之中，我們能擷取來自Dialogflow的資料進行判斷並據此回覆。
在我們的範例中，擷取的資料是「用戶輸入的顏色」這個Intent所擷取的參數「color」。
而Fulfillment所做的事是判斷參數「color」的數值來給予回應。

後續步驟

前往「用戶輸入的顏色」這個Intent的設定頁面。
至頁面最底部的「Fulfillment」，
開啟「Enable webhook call for this intent」
